Known gap: orphaned client API methods client/src/api.js defines calls to /api/tasks (GET/POST/PATCH/DELETE) and /api/purchase-orders (GET/POST), and the UI includes TasksModal.vue, but server/main.py implements no such routes — those calls will 404. purchase_orders.json is loaded by mock_data.py but never served. Likewise, Backlog.vue exists as a view but has no route in the router.

Key Design Decisions

No database

All data loads from JSON at startup and lives in process memory. Fast, zero setup, resets on restart — appropriate for a demo of <10K records.

Centralized filtering

Filters are optional query params; the value "all" means "skip this filter" on both client and server, keeping the two sides symmetric.

No UI framework

Charts are hand-rolled SVG; layout is CSS Grid; styling is a slate/gray design system defined in App.vue. No component or chart libraries.
